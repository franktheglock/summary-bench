# Contributing to Summary Arena

Thank you for your interest in contributing! This guide covers how to add test cases, new categories, and improve the tool.

## Adding New Test Cases

### Step 1: Fork and Clone

```bash
git clone https://github.com/YOUR-USERNAME/summary-arena.git
cd summary-arena
```

### Step 2: Add Test Cases

Edit `datasets/v1/test_cases.json` and add your test cases following this schema:

```json
{
  "test_id": "category-source-NNN",
  "category": "news|code|agentic|meeting|legal|scientific|reviews",
  "input_text": "The full text to be summarized...",
  "gold_summary": "Optional reference summary...",
  "context_length_tokens": 1234,
  "source": "dataset_name or 'public_domain'",
  "prompt_template_key": "category_name"
}
```

### Naming Convention for test_id

- Format: `{category}-{source}-{number:03d}`
- Examples: `news-cnn-021`, `code-py-016`, `meeting-012`

### Step 3: Validate

```bash
cd script
pip install -e .
summaryarena validate --check-dataset  # validates all test cases
```

### Step 4: Submit PR

Create a pull request with:
- **Title**: `[dataset] Add N test cases for {category}`
- **Description**: Source of the data, license compatibility, and why these cases are valuable

## Adding New Categories

1. Add the category name to `VALID_CATEGORIES` in `script/summaryarena/models.py`
2. Add a prompt template in `datasets/v1/prompt_templates.yaml`
3. Add at least 10 test cases
4. Update the README table
5. Submit a PR with title `[category] Add {category_name} category`

## Prompt Template Guidelines

Good prompts should:
- Be specific about desired output format (sentence count, focus areas)
- Include a system prompt establishing the "persona"
- Use `{input_text}` as the placeholder for the actual content
- Not bias toward any particular model's strengths

## Data Licensing Requirements

All contributed data **must** be compatible with open-source redistribution:
- ✅ CC0, CC BY, CC BY-SA, Apache 2.0, MIT
- ✅ Public domain, original content
- ✅ HuggingFace datasets with permissive licenses
- ❌ No copyrighted content without explicit permission
- ❌ No data from paywalled sources
- ❌ No content generated by closed-source models (unless you have redistribution rights)

Include the `source` field in each test case to track provenance.

## Code Contributions

### Setup

```bash
cd script
pip install -e ".[dev]"
```

### Code Style

- Python 3.11+ with type hints everywhere
- Format with `ruff format`
- Lint with `ruff check`
- Test with `pytest`

### Running Tests

```bash
cd script
pytest tests/ -v
pytest tests/ -v --cov=summaryarena  # with coverage
```

### PR Template for Code Changes

```
## What
Brief description of the change.

## Why
Motivation and context.

## Testing
How was this tested?

## Checklist
- [ ] Tests pass (`pytest`)
- [ ] Linted (`ruff check`)
- [ ] Documentation updated if needed
- [ ] No breaking changes to output schema
```
